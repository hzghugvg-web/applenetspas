# План работ

Делаю всё вместе, в одном заходе. Ниже — что и как.

## 1. Фикс: закрытие обращения открывает старый чат

Проблема: при клике на «✕» в форме нового обращения нижним слоем ловится клик по карточке уже существующего обращения и открывает чат.

Решение: обернуть оверлей формы в модалку с `pointer-events` изоляцией + подложка (backdrop), которая перехватывает все клики. Крестик — отдельная кнопка над backdrop с `stopPropagation` и на `pointerup`. Гарантирует, что закрытие не «протекает» на карточки списка.

## 2. Редизайн обращений (список + чат)

Список:
- Карточки-«треды» в стиле iMessage: аватар-иконка категории слева, заголовок + последний ответ, справа время и badge непрочитанного.
- Разделение на «Активные» / «Закрытые» через сегмент-контрол сверху.
- Пустое состояние — иллюстрированный блок с CTA «Новое обращение».

Форма нового обращения:
- Полноэкранная модалка снизу (sheet) вместо инлайн-панели.
- Крупный заголовок, категория чипами, поле темы, textarea с автогровом, submit — большая кнопка внизу.

Чат:
- Полноэкранный экран (не оверлей поверх списка).
- Хедер с назад-кнопкой, темой обращения и статусом.
- Пузыри iMessage-стиля: свои — синий градиент справа, админ — серый слева, хвостики, тонкие тени.
- Composer снизу с safe-area, авто-высота, кнопка отправки-стрелка внутри поля.

## 3. Редизайн VPN (направления)

Сейчас: плоские карточки списком.

Станет:
- Верх — «активное направление» крупной картой с флагом-эмодзи страны, пингом (моковый), кнопками «Скопировать» / «Открыть в клиенте».
- Ниже — горизонтальный скролл-каросель с миниатюрами направлений (флаг + название + latency-точка).
- Раздел «Все направления» — сетка 2 колонки на мобилке, карточка = флаг, страна, город, badge протокола (vless).
- Skeleton-загрузка, haptic tap-анимация scale-95.

## 4. Настройки + темы

Настройки:
- Группированный список в стиле iOS Settings: секции с закруглёнными углами, разделители, иконки в цветных квадратах.
- Секции: Профиль, Внешний вид, Уведомления, О приложении, Выход.

Темы (разделены на два переключателя):
- **Режим цвета**: Светлая / Тёмная / Системная — сегмент-контрол.
- **Тема оформления**: 4 темы, каждая меняет primary/accent/шрифты:
  1. **Classic Blue** — текущая, синий #3B82F6.
  2. **Midnight Purple** — фиолет #8B5CF6, глубокие тени.
  3. **Sunset Orange** — тёплый #F97316, мягкие радиусы.
  4. **Forest Mint** — мятный #10B981, минимализм.
- Выбор — карточки-превью с мини-макетом приложения.
- Сохранение в localStorage + применение через `data-theme` атрибут на `<html>` и CSS-переменные в `styles.css`.

## 5. Рассылки

- Убрать кнопку «Открыть сайт» из отображения рассылок (в списке и в баннере). Оставить только «Скопировать».
- В админке при создании рассылки — выбор способа доставки:
  - **Плашка сверху** (top-banner) — тонкая полоска-баннер на всех экранах для всех пользователей, dismissible.
  - **iMessage-стиль** — центрированная модалка-«баннер» как push-уведомление на iPhone (белый/тёмный блюр, иконка приложения, заголовок, текст, кнопка ОК).
- Добавить поле `delivery_style` в таблицу `broadcasts` (`'top' | 'imessage'`), рендерить в `BroadcastBanner` соответствующий UI.

## Технические детали

- Миграция БД: `ALTER TABLE broadcasts ADD COLUMN delivery_style text DEFAULT 'imessage'`.
- Новые CSS-переменные тем в `src/styles.css` через `[data-theme="..."]` селекторы.
- Хук `useTheme()` (colorMode + themeName) с persist в localStorage.
- Framer Motion для sheet-модалок и iMessage-баннера (spring, damping ~25).
- Компоненты: `ComplaintSheet`, `ComplaintChat`, `VpnHero`, `VpnCarousel`, `ThemePicker`, `ColorModeToggle`, `iMessageBanner`, `TopBanner`.
- Все pointer-events на overlay через `createPortal` в `document.body` — исключает bleed-through на список.
